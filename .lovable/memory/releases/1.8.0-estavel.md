# Release 1.8.0 ESTÁVEL — 17/07/2026

Versão selada e publicada em app.clube23.com.br.

## Escopo consolidado desde a 1.7.4
- **Meios de pagamento CRUD** — dono cadastra/edita/exclui meios personalizados (Vale Alimentação, Vale Refeição, etc.); `is_sistema` protege PIX/Dinheiro/Cartões/Cashback/Fiado; RPC `delete_meio_pagamento` bloqueia exclusão com uso; Fiado configurável no bloco de Cashback.
- **Módulo Fiscal (arquitetura de adapters)** — `config_fiscal`, `notas_fiscais`, `notas_fiscais_itens`, `manifestacoes_destinatario`, `produtos_fornecedor`; certificado A1, campos NCM/CFOP/CST/PIS/COFINS em `products`; NFC-e disparada não-bloqueante em `PaymentDialog`/`BalcaoView`; interruptor mestre (ativo/inativo) por empresa; provedor inicial Tecnospeed; troca de adapter (SaaS↔ACBr↔Nativo) sem mexer no Caixa.
- **Manifestação do destinatário** — `ManifestacaoView` para NF-es recebidas pelo CNPJ.
- **Fiscal e Pagamentos movidos para /admin** (grupo Empresa); removidos do /caixa.
- **iFood + Delivery Próprio (Fase 1)** — tabelas `ifood_merchants/event_log/produto_map`, `entregadores`, `entregas`, `entregador_sessoes`; `orders.canal_venda` (PWA/CAIXA/MESA/IFOOD/TELEFONE); `products.preco_ifood` + variações/adicionais; `empresas.markup_ifood_percentual` + RPC `apply_ifood_markup`; CRUDs Entregadores e Lojas iFood; Painel Kanban de Entregas no /caixa. Integração real (OAuth/polling) pausada aguardando credenciais.
- **Framework de Relatórios** — `ReportShell` A4 com cabeçalho (logo/título/pág X de Y), rodapé (razão social/CNPJ/endereço), colunas selecionáveis, filtros, fontes/tamanhos configuráveis, totais monetários, contagem, exportação CSV, orientação Retrato/Paisagem.
- **Relatório Clientes cadastrados** — primeiro relatório operacional com filtros por cidade/bairro/status/fiado/datas.
- **Assistente IA de Relatórios** — chat em /admin → Relatórios; server function `generateReportSpec` via Lovable AI Gateway (openai/gpt-5.5, structured output); whitelist rígida de fontes (clientes, vendas, produtos_estoque) escopada por tenant via `requireSupabaseAuth`; `ReportSpec` executado por `runReportQuery`; gráficos Recharts (bar/line/pie/area); `relatorios_salvos` (RLS `can_manage_empresa`) permite salvar/reutilizar.
- **Impressão simplificada** — removida camada WebUSB/Serial; volta ao `window.print()` da fila do SO com roteamento por setor mantido.

## Segurança (correções aplicadas nesta faixa)
- Defaults `empresa_id` trocados de UUID fixo para `current_empresa_id()` em `categories`, `orders`, `products`, `profiles`, `regras_combos`.
- Políticas `config_fiscal`, `manifestacoes_destinatario`, `notas_fiscais`, `notas_fiscais_itens`, `produtos_fornecedor` migradas para `can_manage_empresa()`.
- Auditadas e ratificadas: `config_pagamentos`, `entregadores`, `notas_fiscais.certificado_senha` (já isoladas por tenant/dono).

## Compromissos preservados
- Motor financeiro e RLS não foram alterados (mem://constraints/motor-financeiro-protegido).
- Todo o ciclo iFood/Delivery/Fiscal/Relatórios é multi-tenant por `empresa_id` + `can_manage_empresa`.
